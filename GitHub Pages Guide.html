CRH Dashboards — Publishing to the Web with GitHub Pages

Turn your HTML files into real websites with a free URL  ·  No server required  ·  Free hosting

PART 1 — What is GitHub Pages?

GitHub is a free platform for storing code. GitHub Pages is a free feature that turns any HTML file in your GitHub account into a live website with a real URL like https://your-username.github.io/crh-wfm/. No server, no hosting cost, no technical setup beyond this guide.

FeatureBefore (local file)After (GitHub Pages)
Open on any device❌ Only your computer✅ Any device, anywhere
Share with a link❌ Must send the file✅ Share a URL
Always up to date❌ Must re-send file✅ Update once, everyone gets it
CostFreeFree
Requires a serverNoNo

PART 2 — Step 1: Create a Free GitHub Account

1

Go to github.com in Chrome.

2

Click Sign up (top right).

3

Enter: email, a username (e.g. crh-callcenter), password.

4

Verify your email address when prompted.

5

On the plan selection screen — choose Free (the default).

ℹ️ Your GitHub username becomes part of your website URL. Choose something professional like crh-callcenter or crhwfm.

PART 3 — Step 2: Create a Repository (Your Project Folder)

1

After signing in, click the + icon (top right) → New repository.

2

Set the following:

FieldValue
Repository namecrh-wfm

3

Click Create repository.

⚠️ The repository must be Public to use GitHub Pages for free. Your HTML files will be visible publicly — this is safe because the dashboards require login and all data stays in Google Sheets (not in the HTML file).

PART 5 — Step 4: Enable GitHub Pages

1

In your repository, click Settings (top tab with gear icon).

2

Scroll down to the Pages section in the left sidebar.

3

Under Branch, select main from the dropdown.

4

Leave the folder as / (root).

5

Click Save.

6

Wait 1–2 minutes.

7

A green banner will appear: "Your site is published at https://crh-callcenter.github.io/crh-wfm/"

✅ Your dashboards are now live! The URLs will be:
  • https://crh-callcenter.github.io/crh-wfm/ → WFM Dashboard (index.html)
  • https://crh-callcenter.github.io/crh-wfm/manager-dashboard.html
  • https://crh-callcenter.github.io/crh-wfm/quality-dashboard.html

PART 6 — Step 5: Update a File When You Make Changes

Whenever you update index.html or any dashboard:

1

Go to github.com → sign in → open your crh-wfm repository.

2

Click on the file you want to update (e.g. index.html).

3

Click the pencil icon ✏ (Edit this file) — or click … → Upload files to replace the whole file.

4

Scroll down → click Commit changes.

5

Wait 1–2 minutes → your live website updates automatically.

Alternative (faster): Upload method

1

Click Add file → Upload files.

2

Drag the updated file — it will replace the old one automatically.

3

Commit → done.

PART 7 — Custom Domain Name (Optional)

If you want a proper URL like dashboard.crh-hospital.ae instead of github.io:

1

Purchase a domain from a registrar (GoDaddy, Namecheap, etc.).

2

In GitHub → Settings → Pages → Custom domain → enter your domain.

3

In your domain registrar → DNS settings → add a CNAME record pointing to crh-callcenter.github.io.

4

Wait 24–48 hours for DNS to propagate.

5

Enable "Enforce HTTPS" in GitHub Pages settings.

PART 8 — Important: SHEET_API Must Be Set Before Publishing

⚠️ Before uploading to GitHub, make sure SHEET_API is already filled in with your GAS Web App URL in all dashboard files. If it is left empty, the dashboards will work in offline mode only — data will not save to Google Sheets.

PART 9 — Troubleshooting

ProblemFix
Site shows "404 Not Found" Wait 5 minutes after enabling Pages, or check that index.html is in the root of the repository (not inside a subfolder).
Site shows old version Hard refresh with Ctrl+Shift+R, or wait 2 minutes for GitHub's cache to clear.
Login works but data doesn't save SHEET_API not set — check the constant at the top of your HTML file and make sure the GAS URL is pasted in correctly.
File upload fails Check file size is under 25 MB. HTML dashboard files are always well under 1 MB, so this should not occur.
GitHub Pages not showing in Settings Make sure the repository is set to Public. Go to Settings → General → Danger Zone → Change visibility.
