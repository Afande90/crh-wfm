CRH WFM Dashboard

Complete Handover & Setup Guide — Czech Rehabilitation Hospital · Call Center · Al Ain, UAE

PART 1 — What You Are Receiving

This package contains a complete Workforce Management Dashboard built as a single HTML file with a Google Sheets backend. No server is needed. Everything runs in the browser and saves data directly to a Google Sheet under your Google account.

Files included

FileWhat it does
index.htmlThe full dashboard — open this in Chrome to use the system
Code.gsThe backend — paste this into Google Apps Script
User Credentials\users.txtAll login usernames and passwords

What the system does

  • Daily attendance tracking (Present / Absent / Late / Leave / Off)
  • Monthly rota planning with auto-generate and Excel import/export
  • Break schedule management (3 slots: 15 min + 30 min + 15 min)
  • Annual leave and shift swap requests with admin approval
  • Live overview dashboard with KPIs
  • Agent personal view (breaks, rota, requests)
  • 19 agents pre-configured across Inbound, Outbound, Supervisor teams

PART 2 — One-Time Setup (Do This First)

⚠️ You only do this setup ONCE. Use your own Google account (Gmail). All data will be saved to your Google Drive.

Step 1 — Create the Google Sheet

1

Go to sheets.google.com and sign in with your Gmail.

Click Blank spreadsheet.

Name it: CRH WFM (click the title at the top to rename).

Step 2 — Add the backend code

1

In the Google Sheet, click the menu: Extensions → Apps Script

2

A code editor will open. Select all existing text (Ctrl+A) and delete it.

3

Open the file Code.gs from the package in Notepad or any text editor.

Select all (Ctrl+A) → Copy (Ctrl+C).

Paste (Ctrl+V) into the Apps Script editor.

4

Click the Save button (floppy disk icon, or Ctrl+S).

Step 3 — Run setup once

1

In the Apps Script editor, find the function dropdown at the top (it may say myFunction or doGet).

Click it and select setup.

2

Click the Run button (▶ play icon).

3

A popup will appear: "Authorization Required".

Click Review permissions → Choose your Google account → Allow.

ℹ️ This allows the script to read/write your Google Sheet. It only accesses this one spreadsheet.
4

A green success popup will appear: "CRH WFM Setup Complete!"

Click OK. Go back to your Google Sheet — you should now see 8 tabs at the bottom: Users, Agents, Attendance, Rota, Breaks, Leave, Swaps, Config.

✅ If you see 8 tabs, setup was successful.

Step 4 — Deploy as Web App

1

In Apps Script, click Deploy → New deployment (top right button).

2

Click the gear icon ⚙ next to "Select type" → choose Web app.

3

Fill in the settings exactly as follows:

SettingValue
DescriptionCRH WFM API
Execute asMe (your Gmail)
Who has accessAnyone
⚠️ "Who has access" must be Anyone (not "Anyone with a Google account"). Otherwise login will fail.
4

Click Deploy.

A URL will appear — it looks like:

https://script.google.com/macros/s/AKfycb.../exec

Copy this URL — you need it in the next step.

Step 5 — Connect the dashboard to your Sheet

1

Open index.html in Notepad++ or VS Code (right-click → Open with).

Press Ctrl+F and search for:

const SHEET_API = '';
2

Replace it with your URL:

const SHEET_API = 'https://script.google.com/macros/s/YOUR_ID/exec';

(Paste your URL between the single quotes.)

3

Save the file (Ctrl+S).

Step 6 — Test the connection

1

Open index.html in Chrome (double-click the file).

Log in as: username yasir / password CRH@Admin2026

2

After login, a green toast at the bottom-right should appear:

✅ Synced from Google Sheets ✓

This means the dashboard is connected and saving data to your Sheet.

PART 3 — How to Update the Web App (After Changes)

ℹ️ Any time Code.gs is updated in the future, you must create a NEW deployment — do not reuse the old one.
1

In Apps Script → Deploy → New deployment

2

Same settings as before: Web app · Execute as Me · Anyone

3

Copy the new URL → update SHEET_API in index.html

PART 8 — Security Notes

  • The Google Sheet is private to your Google account — nobody else can access it unless you share it.
  • The Web App URL acts like a password-protected API — all requests require a valid login token.
  • Change the SECRET value in Code.gs before deploying if you want extra security (then redeploy).
  • Admin passwords should be changed after first login. Update them in the Users sheet tab directly.
  • Never share the Web App URL publicly — treat it like a password.

PART 9 — Transferring Ownership to a New Gmail Account

Use this guide when the current owner's Gmail is leaving the organisation, or you need to move the entire system — Google Sheet, Apps Script backend, and dashboard — to a different Google account.

Phase A — New Owner Creates Their Sheet

1

New owner signs into Google with their own Gmail account.

2

Go to sheets.google.com → click Blank spreadsheet → name it CRH WFM (click the title at the top to rename).

3

Click Extensions → Apps Script → select all existing code (Ctrl+A) → delete it → paste the full contents of Code.gs → save (Ctrl+S).

4

In the function dropdown select setup → click Run (▶) → authorize when prompted → wait for the green "CRH WFM Setup Complete!" popup.

✅ You should now see 8 tabs: Users, Agents, Attendance, Rota, Breaks, Leave, Swaps, Config.
5

Click Deploy → New deployment → gear icon → Web app.

Set: Execute as = Me · Who has access = Anyone → click Deploy.

Copy the new URL — it looks like: https://script.google.com/macros/s/AKfycb.../exec

Phase B — Update the Dashboard

1

Open index.html in Notepad++ or VS Code.

2

Press Ctrl+F and search for const SHEET_API = '.

Replace the old URL with the new one you just copied:

const SHEET_API = 'https://script.google.com/macros/s/YOUR_NEW_ID/exec';
3

Save index.html (Ctrl+S) → open it in Chrome → test login as yasir / CRH@Admin2026.

✅ A green toast "Synced from Google Sheets ✓" confirms the new connection is working.

Phase C — Migrate Existing Data (if any)

1

In the OLD owner's Google Sheet → click the Attendance tab → press Ctrl+A to select all data → Ctrl+C to copy.

2

In the NEW owner's Google Sheet → click the Attendance tab → click cell A2 → press Ctrl+V to paste.

3

Repeat for the following tabs: Rota, Breaks, Leave, Swaps.

4

Users and Config tabs are already seeded by setup() — only copy them if you made custom changes in the old sheet.

Phase D — Remove Old Owner's Access

1

Old owner goes to their Google Sheet → deletes or archives it. Data is now safely in the new Sheet.

2

The old owner's Web App URL is now obsolete — any copy of index.html still pointing to the old URL will fail to connect.

3

Send the updated index.html file (with the new SHEET_API URL) to all managers and users who have a copy.

⚠️ Passwords and agent credentials stay the same — you do NOT need to reset them. The new Gmail only controls where data is stored, not who can log in to the dashboard.
💡 Tip: The dashboard works offline even without Google Sheets. If SHEET_API is left empty, it uses local in-memory data only (not saved on refresh).
