PART 2 — Connect WFM Dashboard to Google Sheets

The WFM Dashboard saves all data to a Google Sheet via a Google Apps Script Web App URL. The Manager Dashboard reads from the same URL. Both must point to the same deployment.

Step A — Get the GAS Web App URL

1

Open your Google Sheet (named CRH WFM) → click Extensions → Apps Script.

2

Click Deploy → Manage deployments (top right).

Copy the Web app URL — it looks like:

https://script.google.com/macros/s/AKfycb.../exec
3

If no deployment exists yet, click Deploy → New deployment → gear icon → Web app → set Execute as = Me · Who has access = Anyone → Deploy → copy the URL.

Step B — Paste the URL into the WFM Dashboard

1

Open index.html in Notepad++ or VS Code.

Press Ctrl+F and search for: const SHEET_API = '';

2

Replace with your URL:

const SHEET_API = 'https://script.google.com/macros/s/YOUR_ID/exec';

Save the file (Ctrl+S).

3

Open index.html in Chrome → log in as yasir / CRH@Admin2026.

✅ Green toast "Synced from Google Sheets ✓" confirms the connection is working.

Step C — Paste the same URL into the Manager Dashboard

1

Open manager-dashboard.html in Chrome.

Click the ⚙ Settings button (top right of the page).

2

Find the WFM API URL field → paste the same GAS Web App URL → click Save.

3

The WFM panel on the Manager Dashboard will now show live attendance, breaks, and rota data.

ℹ️ Both dashboards must use the EXACT same URL. If the WFM owner re-deploys Code.gs, update SHEET_API in both index.html and manager-dashboard.html settings.

PART 3 — Connect Manager Dashboard to xCally

What is xCally?

xCally is the call center's ACD (Automatic Call Distribution) system used at CRH. It routes incoming calls and tracks real-time queue statistics including Service Level, queue wait times, and individual agent status.

xCally API Requirements

SettingWhere to find itExample value
xCally Base URLYour IT department / server IP addresshttp://192.168.1.100 or https://crh-xcally.local
API KeyxCally Admin Panel → Settings → API Keysxcally_key_abc123...
Queue NamexCally Admin → QueuesInbound, Outbound

Steps to Connect

1

Ask your IT team for the xCally server IP address and API key. These are internal credentials — IT controls them.

2

Open manager-dashboard.html in Chrome.

3

Click the ⚙ Settings button (top right corner of the page).

4

Enter the xCally URL (e.g. http://192.168.1.100) and the API key in the respective fields.

Enter the Queue Name(s) you want to monitor (e.g. Inbound).

5

Click Save. The Live Service Level panel will start polling xCally every 30 seconds and display real-time queue stats.

⚠️ The manager dashboard must be opened on a computer connected to the same internal network (LAN) or VPN as the xCally server. It will NOT work from home or from a different network without a VPN connection to the hospital's internal network.

If xCally Blocks the Connection (CORS Error)

A CORS error means the xCally server is rejecting requests from the browser for security reasons. Ask your IT team to take one of the following actions:

1

In xCally admin: Settings → API → Enable CORS → add the dashboard's origin (the folder path or local URL where the dashboard file is opened).

2

Alternatively, ask IT if a xCally Wallboard URL is available — xCally Wallboard mode is designed for browser display and already includes the correct headers.

3

If neither option is available, IT can set up a lightweight reverse proxy (e.g. Nginx) on the same network that forwards requests to xCally with the correct CORS headers.

PART 4 — Quality Evaluation Storage (Google Sheets)

What changed

The Quality Evaluation dashboard no longer stores evaluations in the browser's local storage. All evaluations are now saved directly to the Google Sheet (Evaluations tab) using the same GAS Web App as the WFM dashboard. This means:

  • Evaluations are accessible from any device, any browser
  • The Manager Dashboard reads quality data from Google Sheets automatically
  • No data is lost if you clear the browser or switch computers

Step 1 — Add SHEET_API to the Quality Dashboard

1

Open quality-dashboard.html in Notepad++ or VS Code.

2

Find this line near the top of the script:

const SHEET_API = '';
3

Paste the same GAS Web App URL as index.html:

const SHEET_API = 'https://script.google.com/macros/s/YOUR_ID/exec';
4

Save the file.

Step 2 — Add the Evaluations sheet to Google Sheets (first time only)

The setup() function already creates the Evaluations sheet. If you ran setup() before this update, add it manually:

1

Open your CRH WFM Google Sheet.

2

Click the + button at the bottom to add a new tab.

3

Name it exactly: Evaluations

4

Add these headers in row 1 (one per column):

id | agent | date | callType | callRef | total | bd0 | bd1 | bd2 | bd3 | bd4 | bd5 | notes
5

If you already ran setup() recently, the tab was created automatically — skip this step.

Step 3 — Log in and verify

1

Open quality-dashboard.html in Chrome.

2

Log in as yasir / CRH@Admin2026.

3

Complete and save one evaluation.

4

Open your Google Sheet → click the Evaluations tab.

✅ You should see the evaluation appear as a new row.

Step 4 — Manager Dashboard reads automatically

1

Open manager-dashboard.html.

2

Click ⚙ Settings → make sure the WFM Sheet URL is saved.

3

The Quality panel now reads from Google Sheets automatically — no more "same device" requirement.

ℹ️ The quality dashboard keeps a local browser cache as backup. If Google Sheets is unreachable, it shows the last synced data from the browser cache.
⚠️ After adding SHEET_API, you must create a NEW Web App deployment in Google Apps Script (Deploy → New Deployment) because the Evaluations sheet was added to Code.gs. Copy the new URL into both index.html AND quality-dashboard.html.
